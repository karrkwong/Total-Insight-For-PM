# Total-Insight-For-PM

`Total-Insight-For-PM` 是一个面向产品经理的机会洞察与产品判断写作 skill。

它基于“`5看3定`”框架，用于输出正式书面化的《产品判断书》《方向判断书》或《机会洞察文档》。适合产品方向分析、机会判断、方向突破、立项判断，以及基于研究材料形成明确产品结论的场景。

它不适用于通用 PRD、简单市场综述、普通竞品资料汇总或无证据基础的空泛战略写作。

## 能力概述

这个 skill 可以把分散的研究输入收敛成面向决策的正式文档，核心特征包括：

- 开头直接给出判断结论
- 按 `5看` 展开结构化论证
- 按 `3定` 收束到执行方向
- 明确处理证据边界、假设项与后续补证方向

它适配两类常见工作方式：

- 从零启动研究，在完成最小必要研究后输出最终判断书
- 基于已有研究材料、VOC 摘要或竞品分析，快速收敛成最终判断文档

## 适用场景

- 评估某项 AI 能力是否应进入产品路线
- 判断某个新方向或机会点是否值得做
- 在重大方向判断前完成结构化机会洞察
- 将研究笔记、VOC 摘要、竞品结论整理成可执行的机会洞察文档

## 核心框架

### 5看

- 看行业
- 看用户
- 看竞争
- 看机会
- 看自己

### 3定

- 定控制点
- 定目标
- 定策略

## 输出要求

这个 skill 会强约束输出方式：

- 使用正式中文书面语，而不是聊天式表达
- 文档标题统一采用 `XXXX_机会洞察`
- 第一部分必须是 `执行摘要`
- 主体必须严格采用 `5看` 与 `3定`
- 不得增加框架外的额外章节
- 结论必须可执行
- 至少包含一张对比表或判断矩阵
- 各级标题之间加入分割线，提升扫描效率
- AI 相关内容只描述能力边界、触发机制、输入输出、Agent-Skill 分工与成本约束
- 推测、假设和证据缺口统一进入附录 `备注项`，不混入正文
- 默认采用创新导向，不以“首期”“MVP”“保守收缩”作为默认起点
- 所有创新判断必须有证据支撑，不能编造
- 当命题边界、目标用户、输出要求、证据范围或业务约束不清晰时，必须先补问关键问题，再写最终稿

## 用户与竞品要求

### 看用户必须包含

- VOC
- 场景链路
- 标杆标准
- 用户场景旅程表
- 对应的 Mermaid 用户场景旅程流程图

### 看竞争必须包含

- 直接竞品
- 替代方案与默认路径
- 超越功能数量的链路比较
- 每个核心对标产品至少 `2` 张功能界面截图

## 工作模式

### 1. 从零研究模式

当用户只给出命题或方向时使用。

预期行为：

- 完成行业、用户、竞争三类最小必要研究
- 然后输出最终判断文档
- 如果问题边界仍然模糊，必须先向用户补问关键问题

### 2. 材料收敛模式

当用户已经提供研究材料、VOC 摘要或竞品分析时使用。

预期行为：

- 区分可直接采用的证据、存在不确定性的证据和明显缺口
- 将这些材料收敛为一份判断文档
- 如果用户意图或判断边界仍不清晰，必须先补问关键问题

## 标准文档结构

1. 执行摘要
2. 5看
3. 3定
4. 附录

完整模板见 [references/output_template.md](./references/output_template.md)。

## 证据规则

`Total-Insight-For-PM` 对证据处理有明确约束：

- 优先使用用户提供的材料与一手证据
- 不把建议样本量写成已完成研究
- 不把推测写成事实
- 正文只保留已证实结论
- 假设、证据缺口与后续补证方向统一放入附录
- 创新判断必须有真实数据、样本或事实支持

详细标准见 [references/research_standards.md](./references/research_standards.md)。

## 写作限制

这个 skill 明确避免：

- 聊天式解释口吻
- AI 营销化语言
- “都有价值”“可以继续观察”这类弱结论
- 下列对仗式句型：
  - `不是A，而是B`
  - `不取决于A，而取决于B`
  - `真实价值点不在A，而在B`
  - `优先B然后A`
  - `先做B再做A`
- 默认保守收缩到“首期”“MVP”“先试水”的叙述习惯

## 输出风格

`Total-Insight-For-PM` 适合输出以下类型的文档：

- 判断明确
- 结构稳定
- 可直接进入推进环节
- 对证据边界敏感

它不是为探索性随笔或松散战略发散而设计的。
