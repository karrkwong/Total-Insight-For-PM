# Total-Insight-For-PM 研究标准

## 0. 两种工作模式

- `从零研究模式`：用户只给命题或方向，未提供足够材料。此时应主动补齐行业、用户、竞争的最小必要研究，再进入判断书。
- `材料收敛模式`：用户已给出研究材料、内部结论或 VOC 摘要。此时先做证据分层、缺口识别与判断收敛。
- 两种模式都必须输出判断，不允许只做资料堆叠。
- 默认采用创新导向，不将“首期”“MVP”或保守试水当成默认判断落点；创新判断必须建立在真实证据之上。
- `5看3定`必须围绕同一个主机会点展开：`看行业`、`看用户`、`看竞争`、`看自己`负责形成依据，`看机会`负责收敛，`3定`负责承接。

## 1. 证据等级

按优先级使用：

1. 一手用户证据：访谈、工单、社区原声、评论、销售反馈、可追溯会议纪要。
2. 一手市场证据：官网、发布会、产品页、定价页、官方文档。
3. 二手研究：媒体、研报、第三方总结。

正文仅写已证实内容。推断、假设、证据缺口与待补研究项统一放入附录`备注项`，不在正文做显式标签展示。二手材料不得直接伪装成用户原声。

## 2. VOC 采样要求

- 从零研究模式下，优先覆盖竞品官方社区、Reddit、Quora、抖音、小红书、酷安等与命题相关的平台；平台选择应服从目标用户分布，不机械铺满所有平台。
- 默认目标：每个关键人群至少 `20` 条有效原声；每个关键竞品至少 `30` 条有效用户反馈。
- 若输入仅为聚合结论而非逐条原声，必须明确标记为“主题级证据”，不得宣称已完成逐条 VOC 审校。
- 有效样本至少满足以下条件：
  - 与目标场景直接相关。
  - 能定位来源平台、时间或材料出处。
  - 去重后保留，不把同义转述重复计数。
- 每次输出至少回答四个问题：
  - 样本来自谁。
  - 样本覆盖哪些场景。
  - 样本量是否足以支持排序。
  - 还缺哪类样本会影响判断。
- 需要基于用户原声提炼一张主场景旅程表和一张对应的 Mermaid 流程图，满足以下要求：
  - 从用户进入场景到完成目标形成完整链路。
  - 节点顺序前后逻辑通顺，不跳步、不并置无关节点。
  - 每个节点至少回答四件事：发生了什么、对应核心场景是什么、遇到什么问题或痛点、期望得到什么结果。
  - 节点设计应来自高频用户证据归纳，不得凭空臆造。
  - Mermaid 图必须采用横向阶段卡片式结构：阶段自左向右排列，并对每个阶段固定展示`发生`、`对应核心场景`、`痛点`、`期望`四层内容。
  - Mermaid 图不是孤立代码展示，必须与上方旅程表一一对应，作为旅程表的可视化版本。

## 3. 标杆与竞品矩阵要求

从零研究模式下，至少选取 `3` 个直接竞品或替代解法进入首轮矩阵；若赛道极早期，可补充默认替代路径作为对照组。

每一家核心对标竞品至少补齐 `2` 张功能界面截图，用于校准真实功能形态、入口设计与交互闭环。

竞品矩阵至少包含以下列：

| 列 | 含义 |
| --- | --- |
| 竞品 | 产品或能力名称 |
| 目标用户 | 主要服务对象 |
| 核心场景 | 解决什么关键任务 |
| 入口/触点 | 用户在何处调用 |
| 功能界面截图 | 至少 2 张关键功能界面截图 |
| 能力闭环 | 是否覆盖理解、决策、执行、沉淀 |
| 优势 | 已被验证的强项 |
| 短板 | 未满足的关键问题 |
| 对我方启示 | 应补齐、规避或超越的点 |

不得只比较功能数量，必须比较链路质量、触达路径、稳定性与可信度。

## 4. 机会收敛矩阵要求

`看机会`只用于收敛主机会点，必须从`看行业`、`看用户`、`看竞争`、`看自己`的交集里推出。

机会收敛必须回答六个问题：

- 机会是什么。
- 为什么是现在。
- 为什么是这类用户。
- 为什么现有方案没有完全占住。
- 为什么我们有条件做成。
- 哪些方向不进入主判断集合。

机会点筛选至少经过以下维度：

| 维度 | 判断问题 |
| --- | --- |
| 用户问题强度 | 是否高频、高损失、高意愿解决 |
| 竞争缺口 | 竞品是否仍存在明显空位 |
| 自身可得性 | 现阶段是否具备入口、数据、能力或组织条件 |
| 战略价值 | 是否能形成优先级、差异化或长期防守 |
| 落地难度 | 是否适合当前阶段投入 |

最终只保留 `1-3` 个战略机会点。其余方向应明确归入“当前不纳入主判断集合”或“证据不足暂不成立”。

正式判断书必须明确一个主机会点。若只能列出多个并列方向，视为收敛失败。

`看机会`不得引入前四看没有支撑的新理由。需要补充的新信息进入附录`备注项`或补证任务。

## 5. 自身能力映射要求

至少判断四类能力：

1. 入口或分发控制权
2. 核心能力供给与编排能力
3. 数据、记忆或反馈闭环能力
4. 交付、合规、区域化或商业化支撑能力

若任一关键能力缺失，结论必须同步下调优先级或缩小范围。

若从零研究模式下无法获得真实自身能力信息，应把内部能力判断写成显式假设，不得伪装为已核实事实。

## 6. 反向验证标准

正式判断书、高优先级方向判断、对外汇报稿默认执行反向验证。

### 6.1 角色分工

- 主写 subagent：负责研究、收敛与初稿。
- 红军 subagent：只负责反向验证，输出异议、证据问题、结构问题和表达问题，不改写正文。
- coordinator：负责判断异议是否成立，并统一修订最终稿。

若当前环境无法调用 subagent，coordinator 仍需以独立质检视角执行同等检查。

### 6.2 Loop 上限

- 反向验证最多 `3` 轮。
- 每轮只处理本轮定义的问题，不扩大成新的写作任务。
- 不追求红蓝军完全一致，只判断关键异议是否已经处理到可发布状态。

### 6.3 三轮检查内容

| 轮次 | 检查重点 | 必答问题 |
| --- | --- | --- |
| 第 1 轮 | 同题性与收敛性 | `5看`是否共同指向一个主机会点；`看机会`是否来自前四看；`3定`是否全部服务该机会点 |
| 第 2 轮 | 证据性与判断强度 | 正文是否把推测写成事实；关键结论是否有证据支撑；是否出现无证据的保守收缩或过度创新 |
| 第 3 轮 | 表达纪律与可执行性 | 是否违反禁用表达；是否超出框架；`3定`是否能指导后续产品与策略动作 |

### 6.4 通过标准

- 主机会点明确，且前后章节一致。
- 正文关键结论都有可追溯证据支撑。
- 不存在足以推翻总判断的结构性漏洞。
- 不存在违反写作纪律的高风险表达。
- `3定`可以承接后续产品、研发、商业或组织动作。

### 6.5 失败标准

第 `3` 轮结束后仍存在以下任一问题，宣告正式文档输出失败：

- `5看`与`3定`仍在表达不同机会。
- 主机会点仍无法收敛成一句清晰判断。
- 关键结论仍依赖未证实假设。
- `3定`仍无法回答服务哪个机会点。
- 文档仍明显偏向信息整理，没有形成战略判断。
- 文风或结构仍持续违反硬性约束。

失败时输出失败报告，说明未达成发布条件的原因、卡住层级、已解决问题、未解决问题、需要补充的输入或研究材料，以及是否需要重新定义命题。

## 7. 文风与输出底线

- 使用正式书面语，不写口语化过渡句。
- 不写“AI 很强”“体验很好”之类无判定标准的话。
- 所有优先级、判断、建议都应有证据支撑；假设前提与证据缺口统一放入附录`备注项`。
- 不使用“不是A，而是B”“不取决于A，而取决于B”“真实价值点不在A，而在B”“优先B然后A”“先做B再做A”这类修辞化对仗句。
